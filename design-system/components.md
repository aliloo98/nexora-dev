# Nexora Components v1

## Philosophie

Les composants sont les briques fondamentales de Nexora.

Chaque composant possède un rôle unique.

Un composant ne doit jamais changer de style selon l'écran.

L'utilisateur doit reconnaître immédiatement un composant Nexora.

---

# 1. Priority Card

Rôle

La décision principale.

Il ne peut y en avoir qu'une par écran.

Hiérarchie

Toujours le premier élément important.

Toujours plus grand que toutes les autres cartes.

Contenu

- Badge
- Icône
- Titre
- Explication
- Impact
- CTA principal

Le CTA principal est unique.

---

# 2. KPI Card

Rôle

Afficher un indicateur financier.

Contenu

- Valeur
- Libellé
- Badge d'état
- Variation éventuelle

Le chiffre est toujours l'élément dominant.

Le texte est secondaire.

---

# 3. Reason Card

Titre

Pourquoi ?

Toujours exactement trois raisons.

Chaque raison possède :

- une icône
- un titre court
- une explication

Jamais de longs paragraphes.

---

# 4. Timeline

Rôle

Présenter les événements dans le temps.

Structure

Aujourd'hui

↓

Demain

↓

Cette semaine

↓

Fin du mois

Le jour actuel doit être visuellement dominant.

---

# 5. Goal Card

Contenu

- Progression
- Pourcentage
- Date cible
- Montant restant

La progression doit être immédiatement compréhensible.

---

# 6. Alert

Les alertes ne sont jamais décoratives.

Une alerte existe uniquement si une action est réellement nécessaire.

---

# 7. Badge

Types

- Premium
- Stable
- Attention
- Risque
- Succès

Toujours sobres.

Jamais flashy.

---

# 8. Boutons

Il existe uniquement :

- Primary
- Secondary
- Ghost
- Danger

Aucun autre style n'est autorisé.

---

# Règles

Jamais deux cartes Priorité.

Jamais deux CTA principaux.

Jamais deux cartes dominantes côte à côte.

Toujours une hiérarchie claire.
