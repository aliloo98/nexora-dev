# Nexora Design Roadmap

## Objectif

Cette feuille de route décrit l'ordre officiel des évolutions du Design System et de l'interface de Nexora.

Chaque phase doit être terminée, testée et validée avant de passer à la suivante.

---

# Phase 1 — Fondations ✅

Objectif :

Créer les bases du Design System.

Documents :

- ✅ README.md
- ✅ colors.md
- ✅ components.md
- ✅ ux-principles.md
- ✅ implementation-rules.md

Livrable :

Un Design System documenté, cohérent et prêt à guider les développements.

---

# Phase 2 — Dashboard

Objectif :

Faire converger le Dashboard vers la référence graphique officielle.

Travaux :

- améliorer la hiérarchie visuelle ;
- mettre en avant la Priority Card ;
- harmoniser les KPI Cards ;
- améliorer les espaces et les alignements ;
- appliquer la palette officielle ;
- conserver la logique métier existante.

Validation :

- aucun changement fonctionnel ;
- tous les tests passent ;
- build valide.

---

# Phase 3 — Plan

Objectif :

Transformer le Plan en véritable assistant de décision.

Travaux :

- meilleure hiérarchie ;
- priorité plus visible ;
- timeline plus claire ;
- CTA mieux mis en valeur ;
- cohérence avec le Dashboard.

---

# Phase 4 — Analyse

Objectif :

Mettre les indicateurs financiers au centre de l'expérience.

Travaux :

- meilleure lecture des graphiques ;
- cartes analytiques harmonisées ;
- informations plus faciles à comprendre.

---

# Phase 5 — Transactions

Objectif :

Créer une expérience de saisie rapide, claire et rassurante.

Travaux :

- formulaire simplifié ;
- meilleure lisibilité ;
- catégories plus intuitives ;
- historique optimisé.

---

# Phase 6 — Paramètres

Objectif :

Uniformiser toute la partie configuration.

Travaux :

- sections plus lisibles ;
- meilleure organisation ;
- composants réutilisés.

---

# Phase 7 — Responsive

Objectif :

Optimiser toute l'expérience mobile.

Travaux :

- marges ;
- cartes ;
- navigation ;
- confort tactile.

---

# Phase 8 — Motion

Objectif :

Ajouter des animations utiles.

Principes :

- discrètes ;
- rapides ;
- informatives ;
- jamais décoratives.

---

# Phase 9 — Accessibilité

Objectif :

Rendre Nexora accessible au plus grand nombre.

Travaux :

- contrastes ;
- navigation clavier ;
- lecteurs d'écran ;
- focus visibles ;
- tailles de texte.

---

# Règle finale

Aucune phase ne doit commencer tant que la précédente n'est pas validée.

La qualité est prioritaire sur la vitesse.
