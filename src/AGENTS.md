# Nexora

Ce fichier décrit les règles que toute IA ou tout développeur doit respecter avant de modifier le projet.

---

## Vision

Nexora est un cockpit de décision financière.

Le but n'est pas simplement de suivre un budget, mais d'aider l'utilisateur à prendre de meilleures décisions.

## Règles de développement

- Ne jamais casser la logique métier.
- Une tâche = un objectif.
- Réutiliser les composants existants.
- Respecter le Design System.
- Préserver les performances.
- Maintenir les tests au vert.
- Ne pas introduire de dette technique inutile.

---

## Validation

Avant chaque commit :

- Build réussi.
- Tests réussis.
- Aucun avertissement critique.
- Vérification visuelle.

## Philosophie
